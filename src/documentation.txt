- Introduction
The purpose of our website, Dylan’s Bartending School, is to serve as both an educational resource as well as a business portal. 
Users can learn how to make popular cocktails at home, explore professional recipes, and book a private bartender for events. 
Our website is designed to be appealing visually, have easy navigation, and remain fully responsive across various devices.

- Design Theme and Concept
We chose a professional yet approachable scholarly theme that still reflects the upscale atmosphere of a real cocktail bar. 
The overall aesthetic is modern, slightly moody, and elegant, using a dark-and-light color scheme to make content stand out. 
We incorporated the UHD color palette, specifically UHD Red (#c8102e) as our primary accent color for buttons and active states, and UHD Navy (#1f3a5f) for headings and borders.
This combination creates a premium feeling while staying warm and inviting both beginners and experienced users.

- Stylesheet Implementation
The entire site was built using the Bootstrap 5 framework for layout, responsiveness, and consistent styling. 
Bootstrap’s grid system and utility classes managed most of the structural work. 
Any other custom styling was placed in a single external file named style.css. 
In this style sheet, we imported two Google Fonts; Playfair Display as headings and Poppins within the body. 
Custom rules were also created to ensure images display correctly at different screen sizes without distortion.

- Webpage Breakdown
- - 1. index.html (Home Page)
The landing page features a bold hero section highlighting the “Drink of the Week,” an About Us section, and a trigger for the age verification system. 
It includes a top navigation bar and a sidebar on larger screens. 
The page serves as the main entry point and attention-grabber for the site. 
The index page has keywords that will help us rank better on search engines after publishing. 

- - 2. recipes.html
The recipes.html page serves as the learning hub in our website; it sets the stage for our website to become an authority on all things cocktails. 
Our team organized each cocktail recipe on a Bootstrap card in a responsive grid; they show the drink's name, vibe, ingredients, and instructions. 
Our choice of the responsive grid lets our site scale down and display properly on mobile devices. 
This page also has a search bar that helps you find cocktails quickly, making it easy to use and find cocktails. 
The recipes.html page has all the information you need to make these cocktails.

- - 3. booking.html
This page has the core business call-to-action. 
It features a comprehensive 12-control booking form for private bartender services. 
It is organized into clear sections such as Event Information, Party Preferences, and Contact Information to make it easier for users to follow. 
The form uses the POST method and sends all variables to the PHP test page.

- - 4. submitRecipe.html
This page serves as a secondary call-to-action to engage enthusiastic users. 
It allows visitors to send their own cocktail recipes with the possibility of being featured on the home page. 
The form is divided into two clear sections: “Recipe Information” and “Contact Information." 
Recipe Information has one input for the drink name, and three text areas for ingredients, instructions, and notes. 
Contact Information is identical to the contact information collected from the booking form. 
Submitting the recipe form will also take you to the PHP test page, where all variables are present. 

- JavaScript Explanation
Using the "DRY" (Don't repeat yourself) principles, we avoided hardcoding the same features on each of the four HTML pages. 
Instead, we abstracted these elements into a single JavaScript file (ui.js). 
Then, with event-listeners, our script injects the HTML into the necessary containers, making collaboration and maintenance much easier. 

- Form Explanation
Our primary business call-to-action is the booking form found in booking.html. 
It uses twelve form controls to capture the necessary details for an event booking. 
The breakdown of this form includes a dropdown for event type, time and date inputs, a number input for guest count, radio buttons for glassware rentals, 
multiple checkboxes for preferred base spirits, a text area for custom requests, then inputs for customer names, phone numbers, and emails. 
After clicking submit, the data sends all the variables to the PHP testing server. 

- Conclusion
Building Dylan’s Bartending School was an excellent opportunity to apply and strengthen our HTML, CSS, and JavaScript skills. 
After brainstorming several themes, the group agreed on a bartending school concept. 
Having direct access to a professional bartender in the family allowed us to create an authentic and realistic list of classic cocktails. 
One of our biggest challenges was creating a stylish and consistent layout that worked well on both desktop and mobile without reusing large blocks of code. 
Our team collaborated effectively through weekly Zoom meetings, GroupMe, GitHub, and email. 
We are proud of the final website and believe it successfully meets all project requirements while providing useful and engaging user experience.
